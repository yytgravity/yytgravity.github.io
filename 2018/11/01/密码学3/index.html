ASCI码
ASCII (American Standard Code for Information
Interchange,美国信息交换标准代码)是基于拉丁字母的一套电脑编码系统,主要用于显示现代英语和其他西欧语言。它是现今最通用的单字节编码系统,并等同于国际标准ISO/EC 646。
请注意, ASCII是American Standard Code forInformation Interchange缩写,而不是ASCI(罗马数字2),有很多人在这个地方产生误解。

Hex编码

1.什么是Hex?
将每一个字节表示的十六进制表示的内容,用字符串来显示。
2.作用:
将不可见的,复杂的字节数组数据,转换为可显示的字符串数据
类似于Base64编码算法型: Base64将三个字节转换为四个字符, Hex将三个字节转换为六个。
·应用场景:
在XML,JSON等文本中包含不可见数据(二进制数据)时使用。

RFC4042&UTF9

Request For Comments (RFC) ,是一系列以编号排定的文件。文件收集了有关互联网相关信息,以及UNIX和互联网社区的软件文件。目前RFC文件是由Internet Society (ISOC)赞助发行。基本的互联网通信协议都有在RFC文件内详细说明。RFC文件还额外加入许多的论题在标准内,例如对于互联网新开发的协议及发展中所有的记录。因此几乎所有的互联网标准都有收录在RFC文件之中。在2005年4月1日, RFC4042定义了”UTF-9和UTF-18”的格式以及编码方式。
定义为: “Unicode目前表示格式(UTF-7,UTF-8, UTF-16)不存储和计算有效的平台,利用9位态作为一种天然的存储单元,而不是8位字节。
